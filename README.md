# 수행평가 점수 검산기

일람표(성적 결과 파일)를 업로드하면, **배점 기준(채점 루브릭)에 등록된 평가요소 배점 조합으로는 만들 수 없는 점수**를 찾아내 표시하는 웹앱입니다. 입력 실수나 전산 오류로 들어간 "있을 수 없는 점수"를 성적 제출 전에 걸러내는 용도입니다.

추가로 **검증한 과목명과 결과(일치/불일치)를 누적하는 "관리자 기록" 기능**이 있습니다.

- **점수 데이터는 완전 클라이언트 사이드** — 학생 이름·점수는 브라우저 밖으로 절대 나가지 않습니다.
- **관리자 기록은 선택적으로 백엔드에 집계** — 과목명·결과·건수·시각만 보냅니다(아래 설정 참고).
- **GitHub Pages로 바로 배포** 가능합니다.

---

## 어떻게 검사하나요?

각 수행평가의 총점은 **평가요소마다 배점 단계를 하나씩 골라 더한 값** 중 하나여야 합니다.
예를 들어 평가요소 배점이 `코드 작성 40/30/20`, `문제 해결 30/20/10`, `발표 30/20/10` 이라면, 나올 수 있는 총점은 그 조합 값들(최대 100 … 최소 40)뿐입니다. 이 조합으로 만들 수 없는 점수(예: 39점)가 일람표에 있으면 **"불가능 점수"** 로 표시합니다.

---

## 사용 방법 (점수 검산)

1. `index.html` 을 브라우저에서 엽니다(또는 배포한 GitHub Pages 주소).
2. 상단 **"점수 검산"** 탭에서 **일람표 파일**(`.xlsx`)을 드래그하거나 눌러서 올립니다. 여러 개 동시 가능.
   - 지원 형식: **교과목별 일람표**(점수 1개 열), **강의실별 일람표**(수행평가 여러 열)
3. 파일별 결과 카드가 나타납니다.
   - 🔴 **불가능 점수** / 🟢 **정상** / 🟡 **확인 필요**(기준에서 못 찾은 수행평가) / ⚪ **미응시·빈칸**
4. **"이상 항목 CSV 내려받기"** 로 발견된 불가능 점수 목록을 받을 수 있습니다.

---

## 관리자 기록 기능

검증을 실행할 때마다 **{과목명, 결과(일치/불일치/확인필요), 점검 건수, 불가능 건수, 시각}** 이 기록됩니다.
상단 **"관리자 기록"** 탭에서 총 검증 횟수·일치/불일치 수·과목별 집계·최근 기록을 표로 확인하고 CSV로 받을 수 있습니다.

- **일치** = 불가능 점수가 하나도 없음 / **불일치** = 불가능 점수 발견 / **확인필요** = 검증 가능한 점수가 없었음(과목·평가명 미매칭 등)
- **학생 이름·점수는 전송하지 않습니다.** 과목명과 결과 집계만 기록됩니다.

### 동작 모드 두 가지

1. **세션 데모 모드 (기본값)** — `index.html` 에 백엔드 주소를 넣지 않은 상태.
   기록이 **그 브라우저 세션에만** 쌓이고 새로고침하면 사라집니다. 기능을 바로 체험할 때 유용합니다.
2. **백엔드 연결 모드** — 아래처럼 백엔드를 연결하면, **모든 사용자의 검증 기록이 한곳(구글 시트)에 실시간 누적**되고, 관리자 탭의 "새로고침"으로 전체 기록을 불러옵니다.

> ⚠️ GitHub Pages는 정적 호스팅이라 **그 자체로는 여러 사용자의 데이터를 모으는 공용 저장소가 될 수 없습니다.** 실시간 누적에는 아래의 무료 백엔드(구글 시트)가 필요합니다.

### 백엔드 연결 (구글 시트 + Apps Script · 무료)

함께 제공된 **`backend_apps_script.gs`** 파일을 사용합니다.

1. 구글 드라이브에서 **새 구글 시트**를 만듭니다(이 시트가 곧 "관리자 DB"가 됩니다).
2. 시트 메뉴 **[확장 프로그램] → [Apps Script]** 를 엽니다.
3. 기본 코드를 지우고 **`backend_apps_script.gs` 전체**를 붙여넣고 저장합니다.
4. 오른쪽 위 **[배포] → [새 배포] → 유형 "웹 앱"** 선택 후:
   - **실행: 나(본인)**
   - **액세스 권한: 모든 사용자**  ← 반드시 이렇게
   - **[배포]** → 권한 승인 → **"웹 앱 URL"**(`.../exec`)을 복사합니다.
5. `index.html` 을 열어 상단의 설정 부분을 수정합니다:

   ```js
   const USAGE_LOG = {
     endpoint: "여기에_복사한_웹앱_URL_붙여넣기",   // 예: https://script.google.com/macros/s/AKfyc.../exec
     key: "",          // (선택) 보호용 키 — 사용하려면 .gs 의 SCRIPT_KEY 와 같은 값으로
     adminPass: "",    // (선택) 관리자 탭 진입 암호(간단 보호)
     logFilename: false
   };
   ```
6. 수정한 `index.html` 을 저장하고 GitHub에 올리면 끝. 이제 누가 검증하든 그 과목·결과가 시트에 쌓이고, 관리자 탭에서 한눈에 볼 수 있습니다.

> 기록은 구글 시트의 `log` 시트에 한 줄씩 쌓입니다. **시트를 직접 열어 봐도** 됩니다(엑셀처럼 정렬·필터·피벗 가능).

#### (선택) 보호
- `.gs` 의 `SCRIPT_KEY` 와 `index.html` 의 `USAGE_LOG.key` 를 같은 값으로 정하면, 그 키가 있어야 기록·조회가 됩니다.
- `USAGE_LOG.adminPass` 를 정하면 관리자 탭 진입 시 암호를 묻습니다.
- ※ 정적 사이트의 클라이언트 암호는 **간단한 가림막**일 뿐 강력한 보안은 아닙니다. 민감하지 않은 활용 통계 용도로 사용하세요.

### 대안: Supabase 등 다른 백엔드
같은 계약(아래)을 만족하는 어떤 HTTPS 엔드포인트든 연결할 수 있습니다.
- **기록(POST)**: 본문 `Content-Type: text/plain` 에 JSON `{actionः"log", subject, result, checked, ok, impossible, unverifiable, students, format, ts, key}` 를 받아 저장.
- **조회(GET)**: `?action=list&callback=FN&key=...` 요청에 **JSONP**(`FN([...])`)로 기록 배열 반환.
조회를 JSONP로 받는 이유는 정적 사이트에서 CORS 없이 안정적으로 읽기 위해서입니다. (Supabase처럼 CORS가 열린 REST를 쓰려면 이 부분을 fetch 방식으로 바꾸면 됩니다.)

## 배점 기준(루브릭) 업데이트하기

배점 기준 데이터는 `index.html` 안에 내장되어 있습니다.

### 방법 1 — 앱에서 임시 교체 (가장 간단)
"점수 검산" 탭의 **"배점 기준 파일 교체"** 버튼으로 새 `.xlsx`(또는 `.csv`)를 올리면 즉시 적용되고 올린 일람표가 재검증됩니다. (새로고침하면 사라지는 임시 적용)

### 방법 2 — 파일에 영구 반영 (배포용)
배점 기준 파일은 다음 4개 열을 가진 시트여야 합니다(열 순서 무관, 헤더 이름으로 인식):

| 과목명 | 수행평가명 | 평가요소 | 배점(점) |
|---|---|---|---|
| 정보과학(2015) | 코드 리뷰(100점) | 기능 구현(40점) | 40 |
| 정보과학(2015) | 코드 리뷰(100점) | 기능 구현(40점) | 35 |

같은 (과목·수행평가·평가요소)에 대해 가능한 배점만큼 행을 반복합니다.
영구 내장은 `index.html` 안 `const EMBEDDED_DB = {...}` 부분을 아래 구조의 JSON으로 교체하면 됩니다:

```json
{
  "version": "2026-06-15",
  "source": "과목별_수행평가_평가요소_배점.xlsx",
  "subjects": {
    "정보과학(2015)": {
      "코드 리뷰(100점)": {
        "기능 구현(40점)": [40, 35, 30],
        "발표(30점)": [30, 20, 10]
      }
    }
  }
}
```

## GitHub Pages 배포 방법

1. GitHub에서 새 저장소를 만듭니다.
2. `index.html` 을 업로드하고 **Commit**.
3. 저장소 **Settings → Pages** → Source를 **Deploy from a branch**, Branch를 **`main` / `/ (root)`** 로 두고 **Save**.
4. 잠시 후 `https://<사용자이름>.github.io/<저장소이름>/` 로 접속됩니다.

### 완전 오프라인/사내망
외부 CDN이 막힌 환경이면 SheetJS(`xlsx.full.min.js`)를 내려받아 같은 폴더에 두고, `index.html` 상단의
`<script src="https://cdnjs.cloudflare.com/.../xlsx.full.min.js"></script>` 를 `<script src="xlsx.full.min.js"></script>` 로 바꾸면 됩니다.
(글꼴 `<link>` 줄은 지워도 한글 시스템 글꼴로 정상 표시됩니다. 단, 관리자 기록의 백엔드 연결은 인터넷이 필요합니다.)

## 파일에 대한 참고

- **과목 인식**: "교과목 : …" 항목의 과목명을 배점 기준과 대조하며, 괄호 안 학년/연도 표기(예: `정보과학(2)`)는 무시합니다.
- **수행평가 인식**: `코드 리뷰(만점 100.00, 35.00%)` 같은 열 제목에서 평가명을 읽고, **합계·비고** 열은 검사하지 않습니다.
- 같은 이름의 수행평가가 여러 과목에 있어 특정할 수 없으면 **"확인 필요"** 로 두고 임의 판정하지 않습니다.

- `README.md` — 이 문서

*검산 기준 요약: 각 수행평가의 점수는 평가요소별 배점 단계를 하나씩 골라 더한 값(조합) 중 하나여야 합니다. 그 집합에 없는 점수를 "불가능 점수"로 봅니다.*
